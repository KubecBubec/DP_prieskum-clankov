## Prehlad merge metod z prieskumu

### Core metody pre tuto DP (odporucane)

- `Averaging`:
  - Preco pouzit: najjednoduchsi a najrychlejsi baseline, skoro nulovy tuning.
  - Slabina: casto vacsia interferencia medzi skillmi.
- `Task Arithmetic`:
  - Preco pouzit: jednoduchy a velmi prakticky pristup cez task vectors.
  - Slabina: citlivost na scaling koeficient.
- `TIES-Merging`:
  - Preco pouzit: cielene riesi konflikty medzi parametrovymi updatmi.
  - Slabina: viac heuristik a parameter tuning ako pri TA.
- `DARE`:
  - Preco pouzit: redukuje sum v updateoch, casto dobry trade-off utility vs forgetting.
  - Slabina: moze byt citlive na volbu nastaveni.

### Dalsie metody spominane v clankoch (teoreticky zmysel)

- `Model Soups` (`uniform`, `greedy`, `learned`):
  - Preco pouzit: overena klasikа pre checkpoint averaging.
  - Kedy: ked chces silny, jednoduchy referencny pristup.
- `SLERP/MLERP`:
  - Preco pouzit: hladka interpolacia medzi modelmi, casto stabilnejsia ako linear average.
  - Kedy: najma pri merge 2 modelov.
- `Fisher Merging`:
  - Preco pouzit: vahovanie podla dolezitosti parametrov (teoreticky robustnejsie).
  - Kedy: ked mas auxiliary statistiky a vacsi rozpocet.
- `RegMean`:
  - Preco pouzit: regularizovane averaging, snaha o mensi forgetting.
  - Kedy: ked priorita je zachovat general schopnosti.
- `Gradient-based merge` (napr. `SuperMerge`, uncertainty gradient matching):
  - Preco pouzit: cielenejsia optimalizacia merge kroku, potencialne silnejsi vykon.
  - Kedy: skor pre pokrocilejsi projekt, vyssia vypoctova narocnost.
- `Activation-Informed Merging (AIM)`:
  - Preco pouzit: merge riadeny aktivaciami/saliency, ochrana dolezitych casti base modelu.
  - Kedy: ked chces zamer na stabilitu a jemnejsiu kontrolu merge.
- `Layer-aware` pristupy (napr. `LATA`):
  - Preco pouzit: rozlisuje ulohu vrstiev a aplikuje merge nerovnomerne.
  - Kedy: ked su zretelne rozdielne skills a chces mensiu interferenciu.
- `Data-free interference-guided` (napr. `WUDI`):
  - Preco pouzit: data-free merge s explicitnou snahou minimalizovat interferenciu.
  - Kedy: ked nechces zavisiet od treningovych dat pri merge.
- `Pre-training phase merging` (`PMA`/`EMA`/`WMA` checkpoint averaging):
  - Preco pouzit: zaujimave pre stabilizaciu a efektivitu vo faze pretrainingu.
  - Kedy: skor mimo rozsah beznej DP (vysoka narocnost).

### Prakticke odporucanie rozsahu

- Minimum: `Averaging + Task Arithmetic + TIES + DARE`.
- Optional navyse: `SLERP` alebo `RegMean` ako 5. metoda.
- Na neskor: gradient-based a pre-training metody, ak ostane cas a compute.
